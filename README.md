# 🥗 Lunch Service API

Внутрішній сервіс для співробітників компанії, який допомагає обрати місце для обіду. Ресторани щодня завантажують актуальні меню через API, а співробітники голосують за найкращий варіант через мобільний додаток.

---

## ✨ Основні можливості

*   **Автентифікація**: Безпечний доступ до системи за допомогою JWT-токенів.
*   **Управління ресторанами**: Створення нових закладів та завантаження щоденних меню (доступно лише для адміністраторів).
*   **Система голосування**: Співробітники можуть проголосувати за обране меню один раз на день.
*   **Підтримка версіонування**: Бекенд підтримує різні версії мобільного додатка через заголовок `Build-Version`.

---

## 🛠 Технологічний стек

*   **Backend**: Django + Django REST Framework
*   **Database**: PostgreSQL
*   **Auth**: SimpleJWT (JSON Web Token)
*   **DevOps**: Docker & docker-compose
*   **Testing**: PyTest
*   **Linter**: Flake8

---

## 📦 Швидкий запуск (Docker)

1.  **Клонуйте репозиторій**:
    ```bash
    git clone https://github.com/whynotdimaa/inftest
    cd lunch_service
    ```

2.  **Запустіть контейнери**
    ```bash
    docker-compose up --build -d
    ```

3.  **Запустіть міграції**:
    
    ```bash
    docker-compose exec web python manage.py migrate
    ```

4.  **Створіть суперюзера** (для адмін-панелі та створення ресторанів):
    ```bash
    docker-compose exec web python manage.py createsuperuser
    ```

5.  **Документація**: API Docs доступні за адресою: `http://localhost:8000/api/docs/`

## 📱 Робота з версіонуванням

Мобільний додаток завжди надсилає версію збірки в заголовках запиту. Система автоматично адаптує формат відповіді під конкретного клієнта.

---

*   **Header**: `Build-Version` — назва заголовка для передачі версії.
*   **Значення**: `1` або `2` — підтримувані версії додатка
*   **v1 (Legacy)**: Повертає базову інформацію: назву ресторану та кількість голосів
*   **v2 (Current)**: Повертає розширений об'єкт з `menu_id` та переліком страв `items`

## 🛰 Приклади API запитів (Postman / cURL)

Нижче наведено основні ендпоінти для взаємодії з системою

---

*   **Автентифікація**: **POST** `/api/auth/login/` — отримання JWT-токенів доступу.
*   **Створення меню**: **POST** `/api/restaurants/{id}/menu/` — завантаження страв (Admin Only)
*   **Отримання результатів**: **GET** `/api/voting/results/today/` — результати за сьогодні
*   **Параметри v2**: Використовуйте заголовок `Build-Version: 2` для отримання детальних даних

## 🏗 Архітектура та принципи розробки (SOLID)

Проєкт побудований з урахуванням масштабованості та чистоти коду

---

*   **Services**: Вся бізнес-логіка винесена в окремий шар сервісів у `apps/voting/services.py`
*   **Clean Code**: Кожна функція виконує одну конкретну задачу для кращої читабельності
*   **Документація**: Налаштовано Swagger (`/api/docs/`) для зручної роботи команди
